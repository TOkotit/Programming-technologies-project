# Техническое задание на проект: Freelance Cashflow Predictor

## 1. Цель проекта
Разработка простого веб-приложения для учёта доходов фрилансера. Сервис позволяет сохранять инвойсы, импортировать их из CSV, строить визуализацию исторических поступлений и прогнозировать месячный cashflow на основе прошлых данных.

## 2. Роли пользователей
- Гость: регистрация и просмотр публичной информации.
- Зарегистрированный пользователь: управление своими инвойсами, импорт CSV, просмотр дашборда и экспорт данных.
- Администратор: управление моделями через Django Admin.

## 3. Модели данных
- Client: owner (ForeignKey → User), name, email, notes
- Project: owner (ForeignKey → User), client (ForeignKey → Client), title, description, hourly_rate
- Category: owner (ForeignKey → User), name
- Invoice: owner (ForeignKey → User), project (ForeignKey → Project), category (ForeignKey → Category), date, amount, paid, description, external_id

## 4. Ключевой функционал (User Stories)
- Как зарегистрированный пользователь, я могу добавлять/редактировать/удалять инвойсы и связанные сущности (clients, projects, categories).
- Как пользователь, я могу импортировать CSV с инвойсами; система валидирует строки и создаёт/связывает клиентов и проекты при необходимости.
- Как пользователь, я могу просматривать список инвойсов с поиском и пагинацией.
- Как пользователь, я вижу дашборд с исторической динамикой доходов и прогнозом на N месяцев.
- Как пользователь, я могу экспортировать свои инвойсы в CSV.

## 5. Интеграции и аналитика
Проект поддерживает два варианта аналитики:
- **Минимальный**: прогноз выполняется встроенной функцией (простая регрессия на Python, без внешних тяжёлых библиотек) — подходит для ограничения хостинга.
- **Полный (опционально)**: использование Pandas + scikit-learn для подготовки данных и построения модели, визуализация — Plotly или Chart.js.

## 6. Критерии соответствия требованиям курса
- Реализовано минимум 3 пользовательские модели.
- Используется Django ORM с примерами агрегаций (Sum, Count, Avg).
- Есть формы (ModelForms) с серверной валидацией.
- Есть импорт CSV (веб-форма и management command) и экспорт в CSV.
- README и TZ находятся в корне репозитория, репозиторий публичный.

## 7. Примечания по деплою
Проект призван корректно работать на PythonAnywhere; при необходимости используется лёгкий вариант аналитики без Pandas/scikit-learn для уложения в квоту диска/памяти.
